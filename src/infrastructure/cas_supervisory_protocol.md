# CAS-SPP: Cryptographic Supervisory Proof Protocol
## Institutional Standard for G-SIFI Control Assurance

## 1. Overview
The **Control Assurance Specification (CAS)** defines the formal requirements for any AGI safety control. The **CAS-SPP** protocol provides the cryptographic machinery to prove that these controls were executed correctly in an air-gapped environment (Nitro Enclave).

## 2. Technical Stack
*   **SR-DSL (Supervisory DSL):** A high-level language used by regulators to define "Meta-Invariants."
*   **Compiler Pipeline:** SR-DSL -> [LexAI-Compiler] -> WASM (Runtime) + R1CS (zk-circuit).
*   **Proof Generation:** Groth16 or PlonK proofs generated by the Sentinel sidecar.

## 3. Protocol Workflow
1.  **Commitment:** The sidecar commits the Model Weights hash, OPA Policy hash, and RCE Context hash to the Kafka WORM log.
2.  **Execution:** The AGI model executes within a Nitro Enclave, monitored by a WASM-compiled SR-DSL kernel.
3.  **Proof Generation:** Upon completion, the sidecar generates a zk-SNARK proof ($P$) asserting:
    *   The model output was generated using the committed hashes.
    *   No SR-DSL invariants (e.g., "Non-Escapement") were violated during the execution trace.
4.  **Verification:** Regulators verify $P$ using the public verification key without needing access to the bank's VPC or model weights.

## 4. G-SIFI Control Categories
| CAS ID | Control Name | Proof Target |
| :--- | :--- | :--- |
| **CAS-01** | Epistemic Alignment | Delta between reported intent and internal representation. |
| **CAS-02** | Disparate Impact | Mathematical proof that demographic parity stays in range. |
| **CAS-03** | SEV-0 Readiness | Heartbeat proof of functional hardware kill-switch. |

## 5. Security: PQC & QKD
*   **ML-KEM-1024:** Used for encapsulating the control session keys.
*   **QKD Telemetry:** Quantum-secured distribution of public keys to the Supervisory Audit Gateway.
